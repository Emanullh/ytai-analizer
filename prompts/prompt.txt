2) “Pseudo-LIWC” simple (determinista, con wordlists internas):
   - pronoun_count (I/you/we/he/she/they + variantes)
   - negation_count (not/no/never/n't + variantes es/en)
   - certainty_count (must/always/definitely + variantes)
   - hedging_count (maybe/might/probably + variantes)
   (No usar LIWC propietario. Solo wordlists pequeñas versionadas en repo.)
3) Congruencia título ↔ transcript (sin LLM):
   - title_keyword_coverage: % de tokens importantes del título presentes en transcript
   - title_keyword_early_coverage_30s:
     - si existen segmentos con startSec/endSec: usar ventanas 0-30s reales
     - si no hay timestamps: fallback a “primer N caracteres” del transcript (documentar).
   Evidencia: guardar lista de tokens del título y cuáles aparecen (audit).
4) (Opcional determinista con embeddings, NO AutoGen):
   - title_transcript_sim_cosine usando embeddings (OpenAI text-embedding-3-small o 3-large).
   - Esto NO usa AutoGen; es un request directo a API de embeddings (si OPENAI_API_KEY existe).

Qué SÍ se calcula CON AutoGen (LLM) y debe ir al prompt del agente:
- promise_type multi-label con scores (taxonomía cerrada):
