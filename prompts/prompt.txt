Tarea: Implementar Thumbnail Vision Features Agent (determinista + LLM vision via AutoGen) integrado al flujo actual de export (sin pasos extra fuera del frontend)

Contexto:
- Export flow asíncrono con SSE (exportJobService -> exportService)
- Ya existe AutoGen runtime del primer agente:

  - apps/api/scripts/autogen_worker.py
  - requirements-autogen.txt y scripts setup ya creados
- Derived por video existe y se mergea:
  exports/<channel>/derived/video_features/<videoId>.json

Objetivo:
Extraer features de miniatura (thumbnail) para cada video exportado y persistirlas en el template reusable.

Salida (template fijo, extendiendo sin romper campos existentes):
En exports/<channel>/derived/video_features/<videoId>.json agregar:
"thumbnailFeatures": {
  "deterministic": { ... },
  "llm": { ... | null },
  "warnings": [...]
}

IMPORTANTE: Qué se calcula SIN AutoGen (determinista) y NO debe ir al prompt LLM
Implementar en TypeScript (Node) y NO llamar LLM para:

A) File/metadata:
- thumbnailLocalPath (relativo ya existe en channel.json)
- fileSizeBytes (fs.stat)
- imageWidth, imageHeight (leer encabezado)
- aspectRatio = width/height

B) OCR determinista (NO LLM):
- Usar tesseract.js (WASM, cross-platform) en apps/api:
  - extraer ocrText (string)
  - ocrConfidenceMean (0..1)
  - ocrBoxes[] con:
    { x, y, w, h, confidence, text } en pixeles
- Derivados:
  - ocrCharCount, ocrWordCount
  - textAreaRatio = sum(boxArea)/imageArea (clamp 0..1)
- Guardar evidencia:
  - ocrBoxes (limit 50: top por confidence o por area)

C) Métricas visuales deterministas (NO LLM):
- brightnessMean (0..1), contrastStd (0..1) sobre luminancia
- colorfulness (Hasler–Süsstrunk) aproximado
- sharpnessLaplacianVar (varianza del Laplacian sobre grayscale)
- edgeDensity (Sobel magnitude > threshold) / pixelCount
Implementación recomendada:
- usar `sharp` para decodificar JPG->raw RGB buffer (o alternativa ya existente)
- cálculos en TS puro sobre el buffer (sin OpenCV)
- downscale a ancho <= 256 para performance (mantener proporción)

D) Congruencia título ↔ OCR (NO LLM):
- thumb_ocr_title_overlap_jaccard sobre tokens normalizados
- thumb_ocr_title_overlap_tokens: {titleTokens[], ocrTokens[], overlapTokens[]} como evidencia
- hasBigText (determinista): textAreaRatio >= 0.08 OR ocrWordCount >= 3

Qué SÍ se calcula CON AutoGen (LLM vision) y debe ir al prompt del agente:
Solo lo que requiere percepción semántica/rostros/emoción/clasificación de estilo:

1) archetype (taxonomía cerrada):
   reaction | diagram | logo | portrait | screenshot | text-heavy | collage | other
2) faceSignals (cerrado y auditable):
   - faceCountBucket: 0 | 1 | 2 | 3plus
   - dominantFacePosition: left|center|right y top|mid|bottom (si hay caras; si no, "unknown")
   - faceEmotionTone: positive|negative|neutral|mixed|unknown
   - hasEyeContact: true|false|unknown
   Todo con confidence 0..1 y evidencia en forma de bboxes NORMALIZADOS (0..1), aunque sean aproximados.
3) clutterLevel (cerrado): low|medium|high + confidence
4) styleTags (multi-label cerrado, max 6):
   high-contrast, low-contrast, colorful, dark, minimal, cluttered, clean, big-text, no-text, face, no-face, logo-heavy, screenshot-like, diagram-like

IMPORTANTE anti-alucinación:
- El LLM debe recibir:
  - la imagen (thumbnail) por multimodal
  - resumen determinista (ocrText truncado + metrics)
- El LLM debe devolver SOLO JSON según schema y con evidence:
  - evidenceRegions[]: [{label, x, y, w, h}] normalizados en [0,1]
  - evidenceSignals[]: referencias a métricas deterministas (fieldName + value)
- Validación post-LLM en Node:
  - clamp bboxes 0..1
  - si faceCountBucket==0, entonces dominantFacePosition debe ser "unknown" y hasEyeContact "unknown"
  - si styleTags incluye "face" y faceCountBucket==0 => quitar tag y warning
