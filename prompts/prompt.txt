Tarea: Enriquecer Fase 1 — capturar metadatos crudos completos desde YouTube Data API v3 en raw/videos.jsonl y raw/channel.json

Contexto:
- analyze devuelve lista de videos con campos mínimos.
- export recibe selectedVideoIds y genera export con transcript y thumbnail.

Objetivo:
Durante export, para cada video seleccionado, obtener y persistir metadatos crudos (YouTube API) que sirven para correlaciones futuras (pacing, duración, engagement proxy).

Requisitos (Criterios de aceptación):
1) raw/channel.json debe incluir “channelStats” (si disponible via channels.list):
   - subscriberCount, viewCount, videoCount, country, publishedAt, customUrl/handle si aplica
2) raw/videos.jsonl debe tener 1 JSON por línea con campos (por video):
   - videoId
   - title
   - description (puede ser truncado opcional; pero guarda al menos 2–4k chars)
   - publishedAt
   - durationSec (convertir ISO8601 duration a segundos)
   - categoryId
   - tags[] (si viene)
   - defaultLanguage, defaultAudioLanguage
   - madeForKids, liveBroadcastContent
   - statistics: viewCount, likeCount, commentCount
   - thumbnails: { default, medium, high, standard, maxres } (url+width+height si viene)
   - thumbnailLocalPath (relativo) + thumbnailOriginalUrl (elige best)
   - transcriptRef:
     - transcriptPath: raw/transcripts/<videoId>.jsonl
     - transcriptSource: captions|asr|none
     - transcriptStatus: ok|missing|error
3) Performance proxies (derivados básicos pero deterministas) en raw/videos.jsonl:
   - daysSincePublish (calculado en export time)
   - viewsPerDay (viewCount/daysSincePublish, handle división por 0)
   - likeRate, commentRate (si viewCount>0)
4) Cuotas/eficiencia:
   - videos.list debe agrupar IDs (batch) para minimizar requests.
   - caching in-memory aceptable durante el job.
5) Compatibilidad:
   - No cambies el response de /analyze si no es necesario.
   - No rompas channel.json actual (puedes mantenerlo “resumen” y dejar raw/videos.jsonl como el completo).
6) Tests:
   - Unit tests para el mapper ISO8601 duration -> seconds.
   - Integration test de export: mock youtubeService para devolver payloads completos y verificar raw/videos.jsonl y raw/channel.json.

Implementación sugerida:
- apps/api/src/services/youtubeService.ts:
  - Añadir getChannelDetails(channelId) -> channels.list snippet/statistics
  - Añadir getVideoDetails(videoIds[]) -> videos.list snippet/contentDetails/statistics
  - Reutilizar cliente HTTP y manejo de errores existente.
- apps/api/src/services/exportService.ts:
  - Antes de transcribir/descargar thumbnail, cargar detalles de videos en batch.
  - Persistir raw/videos.jsonl de forma incremental para no perder progreso si falla un video.
- Sanitización:
  - description puede traer saltos de línea; ok en JSONL.

Entrega:
- raw/videos.jsonl con metadatos completos reales
- raw/channel.json con stats del canal
- tests verdes
- docs actualizados (DATA_CONTRACTS.md)
