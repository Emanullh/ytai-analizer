  "llm": { ... | null },
  "warnings": [...]
}

IMPORTANTE: Qué se calcula SIN AutoGen (determinista) y NO debe ir al prompt del agente
Implementar en TypeScript y NO llamar LLM para:

A) Hook / entrega de promesa (medible):
1) hook_keyword_hit_time_sec:
   - tokens importantes del título (min len>=3, sin stopwords ES/EN)
   - buscar primer segmento (por orden de tiempo) que contiene algún token
   - si hay startSec: usar ese startSec; si no, set null + warning
   - evidence:
     - matchedToken
     - segmentIndex i
     - snippet (subcadena del segmento)
2) title_keyword_coverage:
   - % tokens importantes del título presentes en transcript completo
   - evidence: {titleTokens[], hitTokens[]}
3) title_keyword_early_coverage_30s:
   - si hay timestamps, usar texto de segmentos con start<30
   - si no hay timestamps, usar primer N chars (ej 800) como proxy + warning
4) promise_delivery_30s_score (determinista con embeddings, NO AutoGen):
   - cosine( embed(title), embed(text_first_30s) )
   - usar OpenAI embeddings si OPENAI_API_KEY; modelo default text-embedding-3-large
   - si no hay API key, set null + warning

B) Pacing (sin video):
- wpm_overall:
  - si hay timestamps: totalWords / (durationSec/60) usando:
    - preferir durationSec del metadata video (raw/videos.jsonl) si existe
    - si no existe, usar lastSegmentEndSec si disponible
    - si no, set null + warning
- wpm_0_30, wpm_30_120, wpm_last_30:
  - si hay timestamps y durationSec: calcular por ventanas
  - si no hay timestamps: null + warning
- wpm_variance:
  - varianza por ventanas (0-30,30-120,rest) si disponibles
- segment_length_stats:
  - chars y words por segmento: median, p90 (si segments>0)
- silence_gap_stats:
  - si timestamps confiables: gaps entre endSec y next.startSec: mean, p90, max (clamp >=0)

C) Estructura retórica por reglas + conteos (determinista):
- step_markers_count: patrones ("step 1","paso 1","primero","luego","por último", etc.)
- list_markers_count: ("top","x things","3 reasons","razones", etc.)
- contrast_markers_count: ("but","however","sin embargo","aunque", etc.)
- story_markers_count: ("when","then","al final","entonces", etc.)
Guardar evidence como topMatches[] con {segmentIndex, match, snippet}.

D) Sentimiento / afecto (determinista, lexicon-based):
- sentiment_mean, sentiment_std, sentiment_trend:
  - usar un lexicon simple (EN+ES) versionado en repo (pequeño) y calcular score por segmento
  - trend: slope de score vs tiempo (si timestamps); si no, vs index
- emotion_peaks:
  - usar wordlists pequeñas por emoción (joy/sadness/anger/fear/surprise)
  - devolver top 3 picos con evidence: segmentIndex + snippet + score
No LLM para sentimiento (evitar alucinación).

E) Topic shift count (opcional determinista con embeddings, NO AutoGen):
- topic_shift_count:
  - construir ventanas (ej cada 20s) si timestamps; si no, chunk por N palabras (ej 120)
  - embed cada ventana con embeddings (text-embedding-3-large)
  - contar saltos donde cosine_distance > threshold (ej 0.22) con evidencia (window indices)
Si no hay OPENAI_API_KEY, set null + warning.

Qué SÍ se calcula CON AutoGen (LLM) y debe ir al prompt del agente
Solo lo “interpretativo” que queremos en taxonomías cerradas y con evidencia:

1) story_arc (clasificación cerrada):
   - problem-solution | listicle | timeline | explainer | debate | investigation | tutorial | other
   Con:
   - confidence 0..1
   - evidenceSegments: [{segmentIndex, snippet}] (snippets exactos del input)

2) sponsor_segments (extracción semiestructurada con evidencia):
   - detectar si hay sponsor mid-roll y extraer:
     - { startSec|null, endSec|null, brand, confidence, evidenceSegments:[{segmentIndex,snippet}] }
   El LLM NO inventa marcas: brand debe aparecer literal en snippet.

3) cta_segments (CTA detectada y tipificada, con evidencia):
   - { type: subscribe|like|comment|link|follow|none, confidence, evidenceSegments:[...] }
   Preferir evidence dentro de los primeros/últimos 60s si existe.

IMPORTANTE: Para minimizar tokens y alucinación, el LLM NO recibe el transcript completo.
Debes construir un payload reducido:
- segmentsSample[]:
  - primeros N segmentos (ej 20)
  - + 5 segmentos del medio
  - + últimos 10 segmentos
  Cada texto truncado a 240 chars.
- candidateSponsorSegments[] (determinista por regex “sponsor/brought to you by/patrocinado”):
  segmentos detectados (si alguno)
- candidateCTASegments[] (determinista por regex “subscribe/like/comment/follow”):
  segmentos detectados (si alguno)
Esto se pasa al LLM y el LLM solo confirma/clasifica/extrae con evidencia.
