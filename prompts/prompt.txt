Tarea: Fase 1 — guardar transcript como artefacto por video (raw/transcripts/<videoId>.jsonl) con segmentación mínima y provenance

Contexto:
- channel.json actual guarda transcript como string gigante.
- transcriptPipeline retorna { transcript, status, warning? } y el ASR worker puede emitir eventos.
- Queremos que agentes posteriores puedan trabajar por segmentos y auditar calidad/fuente.

Objetivo:
Guardar, por cada video, un archivo transcript JSONL en raw/transcripts/ con estructura estable y provenance.

Requisitos (Criterios de aceptación):
1) Por cada video exportado, crear:
   raw/transcripts/<videoId>.jsonl
   con líneas JSON del tipo:
   - header (primera línea):
     { "type":"meta", "videoId":"...", "source":"captions|asr|none", "status":"ok|missing|error", "language":"auto|es|en|...", "model": "...", "computeType":"...", "createdAt":"ISO" }
   - segmentos (una o más líneas):
     { "type":"segment", "i":0, "startSec":null|number, "endSec":null|number, "text":"...", "confidence":null|number }
2) Segmentación:
   - Si ASR devuelve segmentos (si puedes obtenerlos del worker), persistir start/end reales.
   - Si captions no trae timestamps, al menos escribir 1 segmento con start/end null y texto completo (o segmentar por párrafos/sentences SIN inventar timestamps).
3) channel.json actual:
   - seguir guardando transcript (string)
   - agregar transcriptSource y transcriptPath (relativo) SIN romper consumidores:
     - si temes compatibilidad estricta, agrega solo en raw/videos.jsonl y deja channel.json igual (pero recomendado añadir exportVersion y campos extra tolerantes).
4) Limpieza:
   - Quitar artefactos obvios del transcript (ej: “New thinking.”) con una función de “sanitizeTranscript” aplicada SOLO al string final si detecta patrones exactos.
   - Guardar en meta si se aplicó cleaning: { transcriptCleaned: true|false }.
5) Observabilidad:
   - Si transcript missing/error, escribir el archivo JSONL igual con meta + warning (sin segments) para que agentes sepan que falta.
6) Tests:
   - Unit tests sanitizeTranscript (casos: no cambia normal, sí quita “New thinking.” exacto al final).
   - Integration export: cuando transcript ok -> existe archivo y tiene meta+segment.
   - Cuando transcript missing -> existe archivo y meta status missing y sin segments.

Implementación sugerida:
- apps/api/src/services/transcriptPipeline.ts:
  - Extender resultado a:
    { transcript, status, warning?, source, language?, asrMeta?, segments? }
  - Mantener compatibilidad con callers (si es breaking, crea función nueva getTranscriptWithArtifacts).
- apps/api/src/services/exportService.ts:
  - Luego de obtener transcript, escribir raw/transcripts/<id>.jsonl.
  - Guardar transcriptPath relativo en raw/videos.jsonl.
- No inventar timestamps para captions.

Entrega:
- raw/transcripts JSONL estable
- channel.json se mantiene compatible
- tests verdes y docs actualizados
