Tarea: Implementar Performance Normalization Agent (100% determinista) integrado al flujo actual de export y a los artifacts derived

Contexto:
- Export genera raw/videos.jsonl (o ya se planea) con fields: viewCount/likeCount/commentCount/publishedAt/durationSec (+ title/desc etc.)
- Derived por video existe: exports/<channel>/derived/video_features/<videoId>.json, merge incremental por agentes previos.
- Objetivo: construir targets normalizados (views/day, engagement rates) y un score residual dentro del canal/timeframe, sin usar YouTube Analytics.

Objetivo:
Para cada video exportado:
1) calcular proxies:
   - daysSincePublish
   - viewsPerDay
   - likeRate, commentRate, engagementRate
2) construir un “performance baseline” por canal (con los videos seleccionados del export):
   - modelo robusto para log_views explicado por edad y duración (+ weekday + isShort)
3) calcular:
   - performanceResidual
   - performancePercentile (percentil del residual dentro del conjunto)
4) persistir:
   - Por video, dentro de derived/video_features/<videoId>.json:
     performance: { daysSincePublish, viewsPerDay, likeRate, commentRate, engagementRate, logViews, residual, percentile }
   - A nivel canal, escribir:
     exports/<channel>/derived/channel_models.json
     con:
     {
       "schemaVersion":"derived.channel_models.v1",
       "computedAt":"ISO",
       "channelId":"...",
       "timeframe":"1m|6m|1y",
       "model": {
         "type":"robust-linear",
         "formula":"log1p(viewCount) ~ log1p(daysSincePublish) + log1p(durationSec) + weekday + isShort",
         "coefficients": {...},
         "intercept": number,
         "fit": { "n": number, "r2Approx": number|null, "madResidual": number, "notes": [...] }
       }
     }
5) todo determinista, sin llamadas de red.

Requisitos / Criterios de aceptación:
- No romper el flujo actual de export ni SSE.
- Si durationSec no está disponible para algún video:
  - usar null en features derivados y excluir ese predictor del modelo o imputar de forma conservadora:
    - isShort = durationSec<=60 (si null, set false y warning)
    - durationTerm: log1p(durationSec) solo si durationSec no null
- Si hay menos de 5 videos seleccionados:
  - NO ajustar modelo; en su lugar:
    - residual=null
    - percentile=null
    - escribir warnings en channel_models.json
- Implementar robust regression:
  Opción A (preferida): Huber loss via iteratively reweighted least squares (IRLS) implementado en TS.
  Opción B: simple OLS + median absolute deviation para residuals (si IRLS es demasiado).
  Debe ser estable y testeable.

Detalles técnicos:
A) Crear módulo:
- apps/api/src/derived/performanceNormalization.ts
  export function computePerformancePerVideo(videos, nowDateISO): { perVideoMap, modelSummary, warnings }
  Donde videos incluye: videoId, publishedAt, viewCount, likeCount, commentCount, durationSec.

B) Helpers deterministas:
- daysSincePublish = max(1, floor((now - publishedAt)/86400000))
- viewsPerDay = viewCount / daysSincePublish
- likeRate = likeCount / max(1, viewCount) (si likeCount missing -> null)
- commentRate = commentCount / max(1, viewCount) (si missing -> null)
- engagementRate = (likeCount+commentCount)/max(1, viewCount) (si ambos missing -> null)
- logViews = ln(1+viewCount)
- weekday: 0-6 (UTC) desde publishedAt
- isShort: durationSec != null && durationSec <= 60

C) Modelo (canal):
- X features:
  - x0 = 1 (intercept)
  - x1 = log1p(daysSincePublish)
  - x2 = log1p(durationSec) (si durationSec disponible; si no, 0 y máscara)
  - x3..x8 one-hot weekday (o sin uno para evitar colinealidad)
  - x9 isShort (0/1)
- Fit:
  - Si implementas IRLS-Huber:
    - iterar 10-20 veces, convergencia por delta coef
  - Si implementas OLS:
    - beta = (X'X)^-1 X'y
    - usar ridge pequeño (lambda=1e-6) si matriz casi singular
- Residual:
  - residual = y - y_pred
  - madResidual = median(|residual - median(residual)|)
- Percentile:
  - percentile = rank(residual)/n (0..1), con tie handling

D) Integración en exportService:
- En apps/api/src/services/exportService.ts:
  - una vez que tengas la lista final de videos exportados (y sus stats/duration):
    - llamar computePerformancePerVideo
    - para cada video: merge performance into derived/video_features/<videoId>.json
    - escribir derived/channel_models.json
  - preferir hacerlo al final (antes de writing_json final) para tener todos los videos.

E) SSE:
- Reusar stage "writing_json" con percent incremental (sin stage nuevo).
- Emitir warning SSE si el modelo no se ajustó (n<5) pero no fallar job.

F) Tests:
