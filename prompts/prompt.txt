Tarea: Implementar cache persistente de export/análisis para reutilizar artifacts y evitar recomputación (sin cambios de UX ni pasos extra en frontend)

Contexto (repo actual):
- Export flow asíncrono con SSE: POST /api/export/jobs + EventSource + stages existentes.
- Export produce (por canal/job) múltiples artifacts en exports/<channel_folder>/:
  - channel.json, manifest.json, raw/*, thumbnails/*, derived/video_features/*, derived/channel_models.json,
    analysis/orchestrator_input.json, analysis/playbook.json, derived/templates.json
- Derived Features Agents se ejecutan en exportService.ts por video.
- Fallos LLM/embeddings no rompen el export; se registran warnings.

Objetivo:
Cuando se ejecuta un nuevo export para el mismo canal/timeframe (y/o subconjunto de videos):
1) Reutilizar lo ya calculado/descargado (thumbnails, transcripts, derived per-video) si siguen válidos.
2) Evitar recomputar trabajo pesado “por video” (captions/ASR, OCR, embeddings, AutoGen classifiers) si existe cache válido.
3) Recalcular solo:
   - lo que depende del conjunto seleccionado o “data cruzada”: performance normalization, cohorts/drivers, orchestrator_input, playbook, templates (porque cambian por selección/timeframe).
4) Mantener el mismo flujo UI (sin nuevos pasos), mismos endpoints, mismos stages. Se permiten más eventos warning/info (pero sin cambiar contratos).
5) El cache debe ser determinista, auditable y seguro (sin path traversal).

Definición de “cache”:
- Cache persistente en filesystem dentro de exports/ usando una carpeta de canal “stable”:
  exports/<channel_folder>/.cache/
- La cache se indexa por:
  - channelId
  - exportVersion (1.1 actual)
  - timeframe
  - videoId
  - fingerprints de inputs relevantes (ej: hash del thumbnail, transcriptSource, model configs)
- Debe soportar “partial reuse”:
  - si el usuario exporta un subconjunto, reutiliza los videos ya cacheados y procesa solo los faltantes.

B) Reglas de validez (cache hits)
Un video se considera “cache hit” para evitar recomputación pesada si:
1) Existen en disco:
   - thumbnailPath
   - raw/transcripts/<videoId>.jsonl
   - derived/video_features/<videoId>.json
2) El CacheEntry coincide con inputs actuales:
   - titleHash/descriptionHash iguales (desde raw/videos.jsonl actual o video details actuales)
   - thumbnailHash igual (hash del archivo jpg) O si falta, recalcular y guardar
   - transcriptHash igual (hash del contenido transcript string o jsonl)
   - asrConfigHash/ocrConfigHash iguales a env/config actual (si cambian, recalcular SOLO esa parte afectada)
   - llmModels/embeddingModel iguales (si cambian, recalcular SOLO los campos que dependan del modelo)
3) Si falta OPENAI_API_KEY o AUTO_GEN_ENABLED=false en la corrida actual:
   - NO invalidar cache existente que ya tenga llm outputs.
   - Simplemente reutilizar lo existente.
4) Si antes llm era null y ahora hay API key, permitir “upgrade”:
   - Recalcular SOLO llm subfields faltantes (titleFeatures.llm, transcriptFeatures.llm, thumbnailFeatures.llm)
   - Sin recalcular deterministic.

C) Integración con exportService (sin endpoints nuevos)
1) En apps/api/src/services/exportService.ts:
   - Antes de procesar cada video, consultar cache:
     - si hit completo -> no re-descargar thumbnail, no re-transcribir, no re-correr OCR, no re-llamar AutoGen, no re-embeddings.
       Solo asegurar que los artifacts estén presentes y listos.
     - si hit parcial -> ejecutar solo las partes faltantes y actualizar derived JSON (merge).
     - si miss -> ejecutar pipeline completo actual.
2) Al finalizar export:
   - SIEMPRE recalcular “cross-video”:
     - performanceNormalization (residual/percentile) depende del conjunto
     - orchestratorDeterministic (cohorts/drivers/exemplars) depende del conjunto
     - orchestratorService (playbook/templates) depende del conjunto
   - Estos outputs se escriben en analysis/ y derived/templates.json (siempre fresh).
3) SSE:
   - Mantener stages actuales.
   - Para cache hits, emitir video_progress stage "writing_json" con percent pequeño y/o warning informativo tipo:
     warning: {videoId, message:"cache hit: reused transcript/thumbnail/derived"}
   - No introducir stage nuevo.
